# Color Alert

Color Alert 是 Windows 10/11 的輕量桌面工具。使用者可以框選最多 10 個監看區域；當區域中穩定出現兩個自訂目標色之一，或同色總面積明顯增加時，程式會播放內建警笛提示音。

## 功能

- 最多同時監看 10 個區域，支援多螢幕、負座標與不同 DPI
- 顏色 1、顏色 2 與每個區域分別追蹤已確認面積
- 以跨螢幕滴管與放大像素格從畫面選取兩個目標色
- 尚未自訂時沿用 `#CEB374` 與 `#6088C5` 作為預設目標色
- 同色已存在時，總面積再次明顯增加仍會形成新的提示事件
- 以獨立的「顏色敏感度」與「面積敏感度」控制近似色容差及新增面積門檻
- 同一取樣週期有多色或多區觸發時，只播放一次提示音序列
- 連續三次取樣確認；穩定下降時自動向下調整後續比較起點
- 可選擇完整播放一次警笛，或間隔 500 ms 完整播放三次
- 可顯示位於擷取範圍外、不擋滑鼠的多區監看框線；已觸發區域顯示橘色
- 每個區域提供「定位」，暫時以閃爍粗框標示清單項目對應的畫面位置
- 最小化或關閉至系統匣，可從選單開始、暫停、新增區域或真正退出
- 區域座標與一般設定保存在 `%LocalAppData%\ColorAlert\settings.json`
- 不連網、不要求管理員權限、不保存螢幕影像

## 系統需求與限制

- Windows 10 或 Windows 11，x64
- 原始碼建置需要 .NET 10 SDK；發布版本為 self-contained，不需另裝 Runtime
- 使用 GDI 擷取一般桌面程式與瀏覽器；不保證支援 DRM 影片、UAC 安全桌面、硬體 overlay 或獨占全螢幕遊戲

## 建置與測試

```powershell
dotnet build ColorAlert.slnx -c Release
dotnet test tests/ColorAlert.Core.Tests/ColorAlert.Core.Tests.csproj -c Release
```

核心偵測測試可在任何支援 .NET 10 的平台執行。WPF、螢幕擷取、警笛播放與多螢幕互動仍須在 Windows 實機驗證。

## 發布單一執行檔

```powershell
dotnet publish src/ColorAlert/ColorAlert.csproj -p:PublishProfile=win-x64
```

輸出位於 `artifacts/win-x64/ColorAlert.exe`。這是免安裝的 Windows x64 單一執行檔。

## 操作方式

1. 啟動程式並按「新增區域」。
2. 拖曳滑鼠框選，按 `Enter` 確認；按 `Esc` 取消。
3. 視需要繼續新增其他區域；可先按每列的「定位」確認畫面位置，再重新選取或刪除。
4. 按顏色 1 或顏色 2 的「從畫面取色」，將十字游標移到目標像素後點一下；按 `Esc` 可取消。
5. 視需要分別調整顏色敏感度、面積敏感度、提示音次數與是否顯示監看區域。
6. 按「開始監看」。暫停／恢復會沿用各區域與顏色目前已確認的面積。
7. 目標色首次出現或相對上次確認面積再次增加並連續三次達標時會提示。

只有系統匣選單的「退出」會真正結束程式。下次啟動會載入上次設定，但不會自動開始監看。
